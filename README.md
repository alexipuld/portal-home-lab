# Portal Home Lab

这是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为个人或团队实验性页面的集中存放与版本管理空间。

## 项目简介

`portal-home-lab` 旨在提供一个轻量的静态页面集合，便于快速部署和分享不同用途的前端页面。  
所有页面均为独立 HTML 文件，无需后端支持，可直接通过浏览器打开或托管于任何静态服务。

## 目录说明

```
/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放独立 HTML 页面
│   └── example.html    # 示例页面
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── style.css
│   └── script.js
└── README.md           # 本文件
```

- `pages/` 目录下每个 HTML 文件可视为一个独立页面，互不依赖。
- `assets/` 存放页面间可能共享的静态资源，也可按需在页面内引用。

## 页面归档说明

- 所有页面均为纯前端实现，不包含服务器端逻辑。
- 页面内容、样式、脚本均内联或引用 `assets/` 下资源。
- 建议每个页面保持自包含，便于单独部署或测试。
- 页面命名建议使用英文小写 + 连字符（如 `my-page.html`），避免特殊字符。

## 维护说明

- 新增页面：在 `pages/` 下创建 `.html` 文件，并遵守命名规范。
- 修改页面：直接更新对应文件，保持与其他页面无冲突。
- 删除页面：移除文件后，若需更新入口 `index.html` 中的链接，也请同步修改。
- 资源管理：若引入新资源，请放入 `assets/` 对应子目录，避免散落。

## 许可

本仓库内容仅供学习和参考使用，无特定许可限制。  
引用第三方资源时请遵守其原作者许可。
